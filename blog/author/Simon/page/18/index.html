What is the Hardware Abstraction Layer or HAL?

The HAL is a part of an Operation System that talks directly to the machine‚Äôs Hardware.

The Windows OS is able to support multiple hardware platforms without having to have different OS versions for each hardware platform thanks to the HAL. During the installation of an OS the HAL is chosen depending on the hardware platform, because of this you are normally unable to remove the HDD from one PC and move it to a PC which has different hardware.

A HAL tip when you are installing VMware VM‚Äôs using¬† Windows Server 2003

Windows 2003 Server unlike Server 2008 doesn‚Äôt have the ability to dynamically adjust the HAL to accommodate extra vCPU‚Äôs after installation. So if you‚Äôre creating a new VM which may possibly need to have an extra vCPU added to it in the future, it is best practice to install the OS on a VM with 2 vCPU‚Äôs as this will install the SMP (Symmetric Multiprocessor) HAL rather than the UP (Uniprocessor) HAL. Once your install of 2003 server is complete you can change your VM‚Äôs vCPU‚Äôs back down to one. You now have the option to add another vCPU whenever its needed.
